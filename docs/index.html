Two-Tier Hybrid Architecture

ivygrep uses a dual-engine architecture: fast indexing with progressive quality upgrades.

1

Hash Index

Sub-second indexing via 128-bit SIMD structural hashes. Instant search availability.

→
2

Hybrid Fusion

Lexical BM25 (Tantivy) and Semantic paths fuse via RRF to return high-precision results.

→
3

Neural Enhancement

Background daemon silently upgrades chunks to quantized ONNX (AllMiniLML6V2Q) with Apple CoreML acceleration on macOS.

🌿 Git-Native Intelligence

ivygrep deeply understands git. This is a core design decision, not an afterthought.

Worktree Overlays

When you git worktree add, ivygrep does not copy the index. It creates a thin overlay storing only divergent chunks & tombstones — referencing the base index by path. A 92K-file repo with 50 changed files creates a ~200 KB overlay instead of duplicating ~50 MB.

Branch-Switch Deltas

Switching branches triggers a targeted Merkle-diff re-index of only the changed files — no full re-scan. Content-based hashes mean byte-identical files across branches are never re-processed.

Supercharge Your AI Agents

ivygrep exposes an MCP (Model Context Protocol) server. Instead of stuffing entire files into your LLM context, your agent can query your codebase natively.
